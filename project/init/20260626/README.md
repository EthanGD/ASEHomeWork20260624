# Windows 一键运行说明

本目录提供两条 Windows CMD 一键脚本，目标是让本地开发时尽量只执行一条简单命令，就完成下面整套流程：

1. `git pull`
2. 同步当前西安本地数据库的表结构和数据到测试环境
3. 启动后端
4. 启动前端

## 一、先做一次配置

第一次使用前，请先复制下面的文件：

```bat
sync_config.sample.ps1
```

复制为：

```bat
sync_config.ps1
```

然后把 `sync_config.ps1` 里的测试环境数据库连接信息改成真实值。

## 二、两条一键脚本

### 1. 从 0 初始化本地数据库，再同步测试环境，并启动前后端

适用场景：
- 第一次启动项目
- 需要清空本地库并重新初始化
- 需要把本地最新表结构和基础数据同步到测试环境

直接运行：

```bat
01_init_db_and_run.cmd
```

这条脚本会自动执行：
- `git pull`
- 编译后端
- 重建本地数据库结构
- 初始化本地基础数据
- 将本地数据库结构和数据同步到测试环境
- 启动后端
- 启动前端

### 2. 保留本地现有数据，直接同步测试环境，并启动前后端

适用场景：
- 本地数据库已经有数据
- 需要保留本地当前数据
- 需要把当前本地数据同步到测试环境后继续开发

直接运行：

```bat
02_run_with_existing_db.cmd
```

这条脚本会自动执行：
- `git pull`
- 编译后端
- 将当前本地数据库结构和数据同步到测试环境
- 启动后端
- 启动前端

## 三、最简单运行方式

在 Windows 资源管理器中直接双击对应 `.cmd` 文件即可。

## 四、默认访问地址

- 前端：`http://localhost:5175/`
- 后端健康检查：`http://localhost:3002/api/health`

## 五、运行前准备

- 已安装 Node.js
- 已安装 Git，且 `git pull` 可正常执行
- PostgreSQL 已启动
- `pg_dump`、`psql` 已安装并加入 `PATH`
- `server/.env` 中本地数据库连接配置正确
- `sync_config.ps1` 中测试环境数据库连接配置正确
